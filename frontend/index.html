How it works
  1. The controller (a Ray driver) splits the image into 128×128 tiles and submits one Ray task per tile to the cluster.
  2. Ray spreads the tasks across worker pods. Too few workers → tasks queue → KubeRay autoscales, asking GKE to add worker pods on Spot VMs (created on demand).
  3. Each finished tile streams back over SSE and is painted here, tagged with the pod that computed it (see the cluster map).
  4. After ~1 min idle, workers scale back to zero.
